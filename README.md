# Projectile Trajectory Control System
# What is this?
I built this project for a course assignment to better understand how control systems work in real life. The goal was to simulate a projectile launcher and see if I could make it hit a target more accurately by adjusting its trajectory through feedback.  
# Why I built it
I wanted to bridge the gap between the math I was learning in class and actually seeing it happen on my laptop. It was one thing to see the equations on paper, but it was much harder (and more interesting) to try and model it in Simulink.  
# How it works
I used MATLAB/Simulink to create a model of the launcher.  I had to figure out how to set up the feedback loops so the system would "self-correct" its aim.  I wrote scripts to run the simulations so I could test how different parameters affected the projectile's path.  
# Challenges I faced
To be honest, it wasn't easy. The hardest part was tuning the parameters so the simulation didn't just crash or go off in a random direction. I spent a lot of time debugging my block diagrams and reading the documentation to figure out why my output wasn't matching the target.  
# What I learned
This project taught me that building something is usually way messier than it looks in a textbook. It forced me to be organized with my files and helped me get much better at using MATLAB for actual problem-solving.
